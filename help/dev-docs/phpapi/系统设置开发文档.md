# 系统设置开发文档

## 1. 文档概述

### 1.1 文档目的
本文档旨在详细说明CRMEB系统中系统设置模块的设计架构、核心功能、代码实现及开发规范，为系统开发人员提供完整的开发参考。

### 1.2 适用范围
本文档适用于CRMEB系统开发人员、维护人员及二次开发人员，用于指导系统设置模块的开发、扩展和维护工作。

### 1.3 术语定义
| 术语 | 解释 |
|------|------|
| RBAC | 基于角色的权限控制（Role-Based Access Control） |
| 管理员等级 | 管理员的层级关系，上级可管理下级 |
| 角色 | 权限的集合，用于分配给管理员 |
| 菜单权限 | 控制管理员可访问的菜单 |
| 操作权限 | 控制管理员可执行的操作 |
| 配置项 | 系统的各项设置参数 |
| 云存储 | 用于存储图片、文件等资源的云服务 |
| 定时任务 | 系统自动执行的任务 |

## 2. 核心功能模块

### 2.1 管理员管理
- **管理员列表**：展示管理员基本信息，支持搜索、分页
- **管理员添加**：添加新管理员，设置基本信息、角色、等级等
- **管理员编辑**：修改管理员信息，包括角色、等级、状态等
- **管理员删除**：删除管理员账号
- **状态管理**：启用/禁用管理员账号
- **密码重置**：重置管理员登录密码
- **权限继承**：基于等级的权限控制，上级可管理下级

### 2.2 角色管理
- **角色列表**：展示角色基本信息，支持搜索、分页
- **角色添加**：添加新角色，设置角色名称、等级、权限等
- **角色编辑**：修改角色信息，包括名称、等级、权限等
- **角色删除**：删除角色
- **状态管理**：启用/禁用角色
- **权限分配**：为角色分配菜单权限和操作权限
- **权限继承**：基于等级的角色控制

### 2.3 菜单管理
- **菜单列表**：展示菜单树状结构，支持搜索、分页
- **菜单添加**：添加新菜单，设置菜单名称、路径、图标、权限等
- **菜单编辑**：修改菜单信息，包括名称、路径、图标、权限等
- **菜单删除**：删除菜单
- **状态管理**：显示/隐藏菜单
- **权限标识管理**：设置菜单的权限标识
- **批量保存**：支持批量保存菜单信息
- **左侧菜单获取**：提供获取当前管理员可访问菜单的接口

### 2.4 系统配置
- **配置分类管理**：管理系统配置的分类
- **配置项管理**：添加、编辑、删除系统配置项
- **配置编辑**：编辑系统基本配置，如网站名称、LOGO、联系方式等
- **配置状态管理**：启用/禁用配置项
- **配置缓存**：管理配置缓存，提高系统性能

### 2.5 存储配置
- **存储方式管理**：配置系统使用的存储方式，如本地存储、阿里云OSS、腾讯云COS等
- **云存储配置**：配置各云存储服务的参数
- **存储域名管理**：设置资源访问域名
- **存储状态管理**：启用/禁用存储方式

### 2.6 系统日志
- **日志记录**：记录系统操作日志、登录日志等
- **日志查询**：支持按条件查询日志
- **日志分页**：分页展示日志信息
- **日志清理**：清理过期日志

### 2.7 数据备份
- **数据库备份**：备份系统数据库
- **备份管理**：管理备份文件，支持下载、删除
- **数据库恢复**：从备份文件恢复数据库
- **数据库优化**：优化数据库性能

### 2.8 定时任务
- **任务列表**：展示系统定时任务
- **任务添加**：添加新的定时任务
- **任务编辑**：修改定时任务信息
- **任务删除**：删除定时任务
- **任务状态管理**：启用/禁用定时任务
- **任务执行**：手动执行定时任务

### 2.9 系统路由管理
- **路由列表**：展示系统路由
- **路由同步**：同步系统路由到数据库
- **路由权限管理**：为路由分配权限

### 2.10 代码生成
- **代码生成配置**：配置代码生成规则
- **代码生成**：根据数据库表生成模型、控制器、视图等代码
- **生成记录管理**：管理代码生成记录

## 3. 代码结构与设计

### 3.1 架构设计
系统采用分层架构设计，严格遵循MVC设计模式：
```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│    Routes       │     │   Controllers   │     │    Services     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                       │                       │
         └───────────────────────┼───────────────────────┘
                                 │
                         ┌─────────────────┐
                         │     Models      │
                         └─────────────────┘
                                 │
                         ┌─────────────────┐
                         │     Daos        │
                         └─────────────────┘
```

## 5. 开发规范与流程

### 5.1 代码规范
1. **命名规范**：
   - 类名采用大驼峰命名法，如 `SystemAdminServices`
   - 方法名采用小驼峰命名法，如 `getAdminList`
   - 变量名采用小驼峰命名法，如 `rolePermission`
   - 常量名采用全大写，下划线分隔，如 `MENU_TYPE_CATALOG`

2. **代码结构**：
   - 严格遵循分层架构，禁止跨层调用
   - 每个方法职责单一，代码行数控制在50行以内
   - 适当添加注释，说明方法功能、参数含义和返回值
   - 权限相关代码必须添加权限验证

3. **权限控制**：
   - 所有管理接口必须添加权限验证
   - 权限验证通过中间件实现
   - 权限标识必须唯一，格式规范
   - 角色权限必须包含所有下级角色的权限

### 5.2 开发流程
1. **需求分析**：明确系统设置的功能需求和业务逻辑
2. **设计阶段**：设计数据结构、API接口和业务流程
3. **编码实现**：按照分层架构编写代码
4. **单元测试**：编写单元测试，确保功能正确性
5. **集成测试**：进行模块集成测试，测试权限控制等功能
6. **安全测试**：对权限控制、密码加密等进行安全测试
7. **代码审查**：提交代码审查，确保代码质量
8. **部署上线**：部署到生产环境

## 6. 常见问题与解决方案

### 6.1 权限继承问题
**问题**：上级管理员无法管理下级管理员或角色
**解决方案**：
1. 检查管理员等级设置，确保上级等级低于下级等级
2. 检查角色等级设置，确保上级角色等级低于下级角色等级
3. 检查权限分配逻辑，确保上级拥有管理下级的权限

### 6.2 菜单显示问题
**问题**：管理员无法看到某些菜单
**解决方案**：
1. 检查菜单状态，确保菜单已启用
2. 检查菜单是否设置为显示
3. 检查管理员所属角色是否拥有该菜单的权限
4. 检查菜单权限标识是否正确

### 6.3 配置不生效问题
**问题**：修改系统配置后不生效
**解决方案**：
1. 检查配置是否已保存成功
2. 检查配置是否已启用
3. 清理配置缓存
4. 检查配置读取逻辑，确保正确读取最新配置

### 6.4 日志记录问题
**问题**：系统操作没有记录日志
**解决方案**：
1. 检查日志中间件是否已启用
2. 检查日志记录逻辑，确保覆盖所有操作
3. 检查日志表是否有写入权限

### 9.2 权限控制流程图

```
权限控制流程：
1. 用户登录系统，获取token
2. 发起API请求，携带token
3. 权限中间件验证token合法性
4. 获取用户所属角色
5. 获取角色拥有的权限
6. 验证请求的路由是否在权限列表中
7. 验证通过，执行请求；验证失败，返回权限错误
```
