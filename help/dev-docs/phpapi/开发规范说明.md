### 1. Service层命名和使用规范

### 2. DAO层职责划分
```php
// ✅ DAO只负责基础CRUD
class UserDao extends BaseDao {
    public function getById(int $id) {}
    public function updateById(int $id, array $data) {}
}

// Service调用DAO
class UserServices {
    public function complexBusiness() {
        $user = $this->dao->getById($id);
        // 业务逻辑处理
        $this->dao->updateById($id, $data);
    }
}
```

### 3. 异常处理标准
```php
// 使用项目特定异常类
throw new ApiException('错误信息', 400);
throw new AdminException('管理端错误', 500);

// 统一返回格式
return $this->success($data);
return $this->fail('错误信息');
```

## 🔧 工具类使用规范
- 使用 `crmeb\utils\` 下的工具类
- 配置获取使用 `SystemConfigService`
- 缓存操作使用 `CacheService`
- 文件上传使用 `UploadService`

## 📝 代码提交规范
1. 提交前运行测试
2. 提交信息格式：`[类型] 描述`
   - `[feat] 新增用户积分功能`
   - `[fix] 修复订单状态更新问题`
   - `[docs] 更新API文档`
   - `[style] 调整代码格式`
3. 避免大块提交，每个提交专注于一个功能或修复

## 🚧 分支管理
- `master` - 生产环境代码
- `develop` - 开发分支
- `feature/xxx` - 功能开发分支
- `hotfix/xxx` - 紧急修复分支
