## 6. 常见问题与解决方案

### 6.1 通知发送失败
**问题**：通知发送失败，如短信发送失败、模板消息发送失败
**解决方案**：
1. 检查配置是否正确，如API密钥、模板ID等
2. 检查第三方服务是否正常，如短信服务商、微信接口等
3. 查看发送日志，定位具体错误原因
4. 实现重试机制，对发送失败的通知进行重试

### 6.2 通知延迟
**问题**：通知发送延迟，如订单支付后很久才收到通知
**解决方案**：
1. 检查队列配置，确保队列服务正常运行
2. 优化通知发送逻辑，减少处理时间
3. 增加队列消费者数量，提高处理能力
4. 实现异步发送，避免阻塞主流程

### 6.3 通知重复发送
**问题**：同一通知重复发送给用户
**解决方案**：
1. 实现幂等性处理，确保同一事件不会触发多次通知
2. 检查事件触发逻辑，避免重复触发
3. 在通知发送前检查是否已经发送过
4. 实现去重机制，过滤重复通知

### 6.4 通知模板管理混乱
**问题**：通知模板过多，管理混乱
**解决方案**：
1. 对模板进行分类管理，如按通知渠道、按业务场景
2. 建立模板命名规范，清晰反映模板用途
3. 定期清理无用模板
4. 实现模板版本管理，便于回滚
