# 营销活动开发文档

## 1. 文档概述

### 1.1 文档目的
本文档旨在详细说明CRMEB系统中营销活动模块的设计架构、核心功能、代码实现及开发规范，为系统开发人员提供完整的开发参考。

### 1.2 适用范围
本文档适用于CRMEB系统开发人员、维护人员及二次开发人员，用于指导营销活动模块的开发、扩展和维护工作。

### 1.3 术语定义
| 术语 | 解释 |
|------|------|
| 限时秒杀 | 在特定时间段内，商品以低于原价的价格销售 |
| 商品砍价 | 用户邀请好友帮忙砍价，以更低价格购买商品 |
| 多人拼团 | 多个用户联合购买商品，享受更低价格 |
| 优惠券 | 可用于抵扣商品价格的电子券 |
| SPU | 标准化产品单元，如"iPhone 13" |
| SKU | 库存量单位，如"iPhone 13 128G 白色" |
| 开团 | 创建拼团活动 |
| 参团 | 加入他人创建的拼团活动 |
| 助力 | 帮助好友砍价 |

## 2. 核心功能模块

### 2.1 限时秒杀
- **秒杀配置**：设置秒杀时间段、参与商品、秒杀价格
- **秒杀商品管理**：添加/编辑/删除参与秒杀的商品
- **秒杀时间段管理**：设置秒杀活动的时间段
- **秒杀记录**：查看用户秒杀记录
- **秒杀状态管理**：开启/关闭秒杀活动

### 2.2 商品砍价
- **砍价商品配置**：设置砍价商品、初始价格、最低价格、砍价次数
- **砍价活动管理**：添加/编辑/删除砍价活动
- **砍价记录管理**：查看用户砍价记录
- **砍价助力记录**：查看用户砍价助力记录
- **砍价状态管理**：开启/关闭砍价活动

### 2.3 多人拼团
- **拼团商品配置**：设置拼团商品、拼团价格、成团人数、拼团有效期
- **拼团活动管理**：添加/编辑/删除拼团活动
- **拼团记录管理**：查看拼团记录、成团状态
- **拼团规则设置**：设置拼团规则，如是否允许重复参团
- **拼团状态管理**：开启/关闭拼团活动

### 2.4 优惠券管理
- **优惠券模板配置**：设置优惠券类型（满减、折扣）、面值、使用条件、有效期
- **优惠券发放管理**：手动发放、自动发放、批量发放优惠券
- **用户优惠券管理**：查看用户领取的优惠券、使用情况
- **优惠券使用统计**：统计优惠券领取量、使用率、优惠金额
- **优惠券状态管理**：开启/关闭优惠券

### 2.5 积分商城
- **积分商品配置**：设置积分商品、积分价格、库存
- **积分商品管理**：添加/编辑/删除积分商品
- **积分兑换记录**：查看用户积分兑换记录
- **积分规则设置**：设置积分获取、使用规则

### 2.6 预售活动
- **预售商品配置**：设置预售商品、预售价格、定金、尾款支付时间
- **预售活动管理**：添加/编辑/删除预售活动
- **预售订单管理**：查看预售订单、定金支付情况、尾款支付情况
- **预售状态管理**：开启/关闭预售活动

## 3. 代码结构与设计

### 3.1 架构设计
系统采用分层架构设计，严格遵循MVC设计模式：
```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   ActivityModel │     │   ActivityDao   │     │ ActivityServices│
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                       │                       │
         └───────────────────────┼───────────────────────┘
                                 │
                         ┌─────────────────┐
                         │ ActivityController│
                         └─────────────────┘
                                 │
                         ┌─────────────────┐
                         │     Routes      │
                         └─────────────────┘
```

### 3.2 核心文件结构

#### 3.2.1 模型层（Model）
所有营销活动模型均位于 `crmeb/app/model/activity/` 目录下，按活动类型分类：

- **秒杀活动**：
  - `seckill/StoreSeckill.php` - 秒杀商品模型

- **砍价活动**：
  - `bargain/StoreBargain.php` - 砍价商品模型
  - `bargain/StoreBargainUser.php` - 砍价用户参与记录模型
  - `bargain/StoreBargainUserHelp.php` - 砍价助力记录模型

- **拼团活动**：
  - `combination/StoreCombination.php` - 拼团商品模型
  - `combination/StorePink.php` - 拼团记录模型

- **优惠券活动**：
  - `coupon/StoreCoupon.php` - 优惠券模板模型
  - `coupon/StoreCouponIssue.php` - 优惠券发放记录模型
  - `coupon/StoreCouponProduct.php` - 优惠券适用商品模型
  - `coupon/StoreCouponUser.php` - 用户优惠券模型

#### 3.2.2 数据访问层（DAO）
所有营销活动DAO均位于 `crmeb/app/dao/activity/` 目录下，按活动类型分类：

- **秒杀活动**：
  - `seckill/StoreSeckillDao.php` - 秒杀商品DAO

- **砍价活动**：
  - `bargain/StoreBargainDao.php` - 砍价商品DAO
  - `bargain/StoreBargainUserDao.php` - 砍价用户参与记录DAO
  - `bargain/StoreBargainUserHelpDao.php` - 砍价助力记录DAO

- **拼团活动**：
  - `combination/StoreCombinationDao.php` - 拼团商品DAO
  - `combination/StorePinkDao.php` - 拼团记录DAO

- **优惠券活动**：
  - `coupon/StoreCouponDao.php` - 优惠券模板DAO
  - `coupon/StoreCouponIssueDao.php` - 优惠券发放记录DAO
  - `coupon/StoreCouponProductDao.php` - 优惠券适用商品DAO
  - `coupon/StoreCouponUserDao.php` - 用户优惠券DAO

#### 3.2.3 业务服务层（Services）
所有营销活动Services均位于 `crmeb/app/services/activity/` 目录下，按活动类型分类：

- **秒杀活动**：
  - `seckill/StoreSeckillServices.php` - 秒杀商品Services

- **砍价活动**：
  - `bargain/StoreBargainServices.php` - 砍价商品Services
  - `bargain/StoreBargainUserServices.php` - 砍价用户参与记录Services
  - `bargain/StoreBargainUserHelpServices.php` - 砍价助力记录Services

- **拼团活动**：
  - `combination/StoreCombinationServices.php` - 拼团商品Services
  - `combination/StorePinkServices.php` - 拼团记录Services

- **优惠券活动**：
  - `coupon/StoreCouponServices.php` - 优惠券模板Services
  - `coupon/StoreCouponIssueServices.php` - 优惠券发放记录Services
  - `coupon/StoreCouponUserServices.php` - 用户优惠券Services

#### 3.2.4 控制器层（Controller）
管理员API控制器位于 `crmeb/app/adminapi/controller/v1/marketing/` 目录下：

- `StoreSeckill.php` - 秒杀活动控制器
- `StoreBargain.php` - 砍价活动控制器
- `StoreCombination.php` - 拼团活动控制器
- `StoreCoupon.php` - 优惠券控制器

移动端API控制器位于 `crmeb/app/api/controller/v1/activity/` 目录下，提供移动端调用的活动接口。

#### 3.2.5 路由配置
营销活动路由配置位于 `crmeb/app/adminapi/route/marketing.php` 文件中，包含了各种活动的增删改查、状态管理、统计等路由。

移动端路由配置位于 `crmeb/app/api/route/` 目录下，根据API版本分类。

## 5. 开发规范与流程

### 5.1 代码规范
1. **命名规范**：
   - 类名采用大驼峰命名法，如 `StoreSeckillServices`
   - 方法名采用小驼峰命名法，如 `getSeckillList`
   - 变量名采用小驼峰命名法，如 `seckillProduct`
   - 常量名采用全大写，下划线分隔，如 `COUPON_TYPE_FULL`

2. **代码结构**：
   - 严格遵循分层架构，禁止跨层调用
   - 每个方法职责单一，代码行数控制在50行以内
   - 适当添加注释，说明方法功能、参数含义和返回值

3. **数据库操作**：
   - 所有数据库操作必须通过Dao层进行
   - 使用ORM提供的查询构建器，避免直接编写SQL
   - 涉及资金、库存等关键数据操作必须使用事务处理

### 5.2 开发流程
1. **需求分析**：明确营销活动的功能需求和业务逻辑
2. **设计阶段**：设计数据结构、API接口和业务流程
3. **编码实现**：按照分层架构编写代码
4. **单元测试**：编写单元测试，确保功能正确性
5. **集成测试**：进行模块集成测试，测试活动与商品、订单等模块的交互
6. **性能测试**：对高并发场景进行性能测试，如秒杀活动
7. **代码审查**：提交代码审查，确保代码质量
8. **部署上线**：部署到生产环境

## 6. 常见问题与解决方案

### 6.1 高并发处理
**问题**：秒杀、拼团等活动在高并发场景下可能出现超卖、数据不一致等问题
**解决方案**：
1. 使用数据库事务和乐观锁机制
2. 实现库存预扣和释放机制
3. 使用Redis等缓存技术，提高库存操作效率
4. 采用分布式锁，确保数据一致性
