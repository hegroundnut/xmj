# CRMEB 后端系统事件文档说明

## 1. 事件机制概述

CRMEB后端系统采用事件驱动架构，通过事件机制实现模块间的解耦和通信。事件系统允许在不直接依赖的情况下，实现不同模块间的交互和业务逻辑扩展。

### 1.1 核心概念

- **事件(Event)**：系统中发生的特定动作或状态变化，如订单创建、用户注册等。
- **监听器(Listener)**：监听特定事件并执行相应处理逻辑的类。
- **事件触发(Trigger)**：在特定业务逻辑点调用事件，通知相关监听器执行。

## 2. 监听器接口定义

所有监听器必须实现 `crmeb\interfaces\ListenerInterface` 接口，该接口定义了统一的事件处理方法。

```php
<?php
namespace crmeb\interfaces;

interface ListenerInterface
{
    public function handle($event): void;
}
```

### 2.1 接口说明

- **handle方法**：事件处理的核心方法，接收一个`$event`参数，包含事件触发时传递的数据。
- 返回类型：`void`，监听器处理完成后无需返回值。

## 3. 事件定义与注册

事件通过 `app/event.php` 文件进行定义和注册，该文件返回一个包含事件配置的数组。

### 3.1 事件配置结构

```php
return [
    'bind' => [], // 事件绑定
    'listen' => [ // 事件监听
        'EventName' => [
            'Listener1::class',
            'Listener2::class',
        ],
    ],
];
```

### 3.2 事件注册示例

```php
return [
    'listen' => [
        'OrderCreateAfterListener' => [\app\listener\order\OrderCreateAfterListener::class], // 订单创建后置事件
        'UserRegisterListener' => [\app\listener\user\RegisterListener::class], // 用户注册后置事件
        // 更多事件...
    ],
];
```

## 5. 事件触发方式

事件通过 `event()` 函数进行触发，该函数是ThinkPHP框架提供的全局函数。

### 5.1 基本触发方式

```php
// 触发事件
 event('EventName', $eventData);

// 示例：触发订单创建事件
 event('OrderCreateAfterListener', [$order, $group, $uid, $key]);
```

### 5.2 事件数据传递

事件触发时，可以传递任意类型的数据，监听器通过 `$event` 参数接收。数据通常以数组形式传递，包含处理业务逻辑所需的所有信息。

## 6. 自定义事件

除了系统内置事件外，CRMEB还支持自定义事件，通过 `CustomEventListener` 实现灵活的业务扩展。

### 6.1 自定义事件触发

```php
// 触发自定义事件
 event('CustomEventListener', ['event_mark', ['data1' => 'value1', 'data2' => 'value2']]);
```

### 6.2 自定义事件处理

自定义事件由 `app\listener\CustomEventListener` 处理，该监听器从数据库中获取配置的自定义代码并执行。

```php
<?php
namespace app\listener;

use app\services\system\SystemEventServices;
use think\facade\Log;

class CustomEventListener
{
    public function handle($event)
    {
        [$mark, $data] = $event;
        try {
            $list = app()->make(SystemEventServices::class)->selectList(['mark' => $mark, 'is_del' => 0, 'is_open' => 1])->toArray();
            foreach ($list as $item) {
                eval(json_decode($item['customCode']));
            }
        } catch (\Throwable $e) {
            $listener_log_open = config("log.listener_log", false);
            if ($listener_log_open) {
                $date = date('Y-m-d H:i:s', time());
                Log::write($date . '自定义事件错误:' . $e->getMessage(), 'listener');
            }
        }
    }
}
```

## 7. 事件处理流程

1. **事件触发**：在业务逻辑中通过 `event()` 函数触发特定事件。
2. **事件分发**：系统根据事件名称查找对应的监听器列表。
3. **监听器执行**：依次调用每个监听器的 `handle()` 方法，并传递事件数据。
4. **业务处理**：监听器执行各自的业务逻辑，如数据处理、消息发送等。
5. **处理完成**：所有监听器执行完成后，事件处理流程结束。

### 8.1 订单创建事件监听器

```php
<?php
namespace app\listener\order;

use app\jobs\notice\PrintJob;
use app\jobs\OrderCreateAfterJob;
use app\jobs\OrderJob;
use app\jobs\ProductLogJob;
use app\jobs\UnpaidOrderCancelJob;
use app\jobs\UnpaidOrderSend;
use app\services\order\StoreOrderCreateServices;
use app\services\order\StoreOrderStatusServices;
use crmeb\interfaces\ListenerInterface;
use crmeb\services\CacheService;
use crmeb\services\SystemConfigService;
use crmeb\utils\Arr;

/**
 * 订单创建后置事件
 * Class OrderCreateAfterListener
 * @package app\listener\order
 */
class OrderCreateAfterListener implements ListenerInterface
{
    public function handle($event): void
    {
        [$order, $group, $uid, $key, $combinationId, $seckillId, $bargainId] = $event;

        // 订单数据创建之后的商品实际金额计算，佣金计算，优惠折扣计算，设置默认地址，清理购物车
        /** @var StoreOrderCreateServices $orderCreate */
        $orderCreate = app()->make(StoreOrderCreateServices::class);
        $orderCreate->orderCreateAfter($order, $group, $combinationId || $seckillId || $bargainId);

        // 清除订单缓存
        CacheService::delete('user_order_' . $uid . $key);

        // 写入订单记录表
        /** @var StoreOrderStatusServices $statusService */
        $statusService = app()->make(StoreOrderStatusServices::class);
        $statusService->save([
            'oid' => $order['id'],
            'change_type' => 'cache_key_create_order',
            'change_message' => '订单生成',
            'change_time' => time()
        ]);

        // 订单自动取消
        $this->pushJob($order['id'], $combinationId, $seckillId, $bargainId);

        // 下单记录
        ProductLogJob::dispatch(['order', ['uid' => $uid, 'order_id' => $order['id']]]);

        // 小票打印
        PrintJob::dispatch([$order['id'], 2]);
    }

    /**
     * 订单自动取消加入延迟消息队列
     * @param int $orderId
     * @param int $combinationId
     * @param int $seckillId
     * @param int $bargainId
     * @return mixed
     */
    public function pushJob(int $orderId, int $combinationId, int $seckillId, int $bargainId)
    {
        // 系统预设取消订单时间段
        $keyValue = ['order_cancel_time', 'order_activity_time', 'order_bargain_time', 'order_seckill_time', 'order_pink_time'];
        // 获取配置
        $systemValue = SystemConfigService::more($keyValue);
        // 格式化数据
        $systemValue = Arr::setValeTime($keyValue, is_array($systemValue) ? $systemValue : []);
        if ($combinationId) {
            $secs = $systemValue['order_pink_time'] ?: $systemValue['order_activity_time'];
        } elseif ($seckillId) {
            $secs = $systemValue['order_seckill_time'] ?: $systemValue['order_activity_time'];
        } elseif ($bargainId) {
            $secs = $systemValue['order_bargain_time'] ?: $systemValue['order_activity_time'];
        } else {
            $secs = $systemValue['order_cancel_time'];
        }
        // 未支付10分钟后发送短信
        UnpaidOrderSend::dispatchSecs(600, [$orderId]);
        // 未支付根据系统设置事件取消订单
        UnpaidOrderCancelJob::dispatchSecs((int)($secs * 3600), [$orderId]);
    }
}
```

     * 注册完成后置事件
     * @param $event
     * @throws \think\db\exception\DataNotFoundException

    {
        [$spreadUid, $userType, $name, $uid, $isNew] = $event;

        if ($spreadUid) {
            if ($isNew) {
                // 邀请新用户增加经验
                /** @var UserBillServices $userBill */

                $userServices->addBrokeragePrice($uid, $spreadUid);

                // 推广新人 处理自己、上级分销等级升级

            // 记录推广绑定关系

            $res = $userSpreadServices->setSpread($uid, $spreadUid);

                event('CustomNoticeListener', [$spreadUid, ['nickname' => $name, 'time' => date('Y-m-d H:i:s'), 'phone' => $phone], 'spread_success']);
            }
        }

        if ($isNew) {
            // 新人优惠券发送
            /**@var StoreCouponIssueServices $storeCoupon */
            $storeCoupon = app()->make(StoreCouponIssueServices::class);
            $storeCoupon->userFirstSubGiveCoupon((int)$uid);

            // 人人分销开启推广权限
            if (sys_config('brokerage_func_status') && sys_config('store_brokerage_statu') == 2) {
                /** @var UserServices $userServices */

                $userServices->update($uid, ['is_promoter' => 1]);
            }

## 9. 最佳实践

### 9.1 事件命名规范

- 事件名称应清晰表达事件的含义和触发时机，如 `OrderCreateAfterListener` 表示订单创建后的事件。
- 使用驼峰命名法，首字母大写。

### 9.2 监听器设计原则

- **单一职责**：每个监听器应只负责处理一个特定的业务逻辑。
- **无副作用**：监听器处理不应修改原始事件数据，避免影响其他监听器。
- **异常处理**：在监听器中适当捕获异常，避免单个监听器失败影响整个事件处理流程。

### 9.3 性能优化

- **异步处理**：对于耗时的操作，建议使用队列进行异步处理，如发送短信、生成报表等。
- **缓存使用**：合理使用缓存减少数据库查询，提高处理效率。
- **避免循环依赖**：监听器之间不应相互依赖，避免形成循环调用。

## 10. 常见问题与解决方案

### 10.1 事件不触发

**问题**：调用 `event()` 函数后，监听器未执行。

**解决方案**：
- 检查事件名称是否与 `event.php` 中注册的名称一致。
- 检查监听器类是否正确实现了 `ListenerInterface` 接口。
- 检查监听器类是否正确注册到对应的事件中。

### 10.2 监听器执行顺序

**问题**：需要控制监听器的执行顺序。

**解决方案**：
- 在 `event.php` 中，监听器的注册顺序决定了执行顺序。
- 如需更复杂的控制，可以在监听器中手动触发其他事件。

### 10.3 事件数据传递

**问题**：监听器无法获取完整的事件数据。

**解决方案**：
- 确保事件触发时传递了所有必要的数据。
- 使用数组形式传递数据，便于扩展和维护。
- 在监听器中对数据进行适当的验证和转换。
