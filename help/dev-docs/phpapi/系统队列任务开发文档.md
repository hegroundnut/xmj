## 4. 任务基类

### 4.1 BaseJobs

任务基类实现了`JobInterface`接口，提供了任务执行的基础逻辑：

```php
namespace crmeb\basic;

use crmeb\interfaces\JobInterface;
use think\queue\Job;
use think\facade\Log;

abstract class BaseJobs implements JobInterface
{
    /**
     * @param Job $job
     * @param $data
     */
    public function fire(Job $job, $data): void
    {
        try {
            $method = $data['method'] ?? 'doJob';
            if (method_exists($this, $method)) {
                if (!isset($data['data'])) {
                    $res = $this->{$method}();
                } else {
                    $res = $this->{$method}(...$data['data']);
                }
                if ($res) {
                    $job->delete();
                } else {
                    if ($job->attempts() > 3) {
                        $job->delete();
                    } else {
                        $job->release();
                    }
                }
            } else {
                $job->delete();
            }
        } catch (\Throwable $e) {
            Log::error('队列任务执行失败：' . $e->getMessage());
            if ($job->attempts() > 3) {
                $job->delete();
            } else {
                $job->release();
            }
        }
    }
}
```

- **fire**：实现JobInterface接口的方法，处理任务执行、错误捕获和失败重试
- **doJob**：默认的任务执行方法，子类可以重写

## 8. 任务执行流程

1. **任务调度**：通过`dispatch`或`Queue`工具将任务加入队列
2. **任务消费**：队列消费者进程从队列中获取任务
3. **任务执行**：调用任务类的`fire`方法执行任务
4. **结果处理**：
   - 执行成功：删除任务
   - 执行失败：根据重试次数决定是否重新执行
   - 重试超限：将任务移至失败队列（如果配置）

## 10. 最佳实践

- **单一职责**：每个任务只负责一个功能

- **数据完整性**：确保任务执行所需的数据完整
- **错误处理**：妥善处理任务执行中的异常
